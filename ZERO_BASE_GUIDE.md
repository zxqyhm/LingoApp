## 步骤 2：解压文件（1分钟）

### Windows 系统：

1. **找到下载的文件**
   - 文件名：`lingo-client.tar.gz`
   - 位置：通常在"下载"文件夹

2. **解压文件**
   - **方法 A**：右键点击文件 → "全部提取" → 选择提取位置（比如桌面）→ 点击"提取"
   - **方法 B**：双击文件 → 拖里面的 `client` 文件夹到桌面

3. **确认解压成功**
   - 应该能看到一个 `client` 文件夹
   - 里面有 `node_modules`、`src`、`app` 等文件夹

### Mac 系统：

1. **双击下载的文件**
   - 系统会自动解压
   - 解压后会出现 `client` 文件夹

2. **确认解压成功**
   - 应该能看到 `client` 文件夹
   - 里面有多个子文件夹

## 步骤 3：安装 Node.js（5-10分钟）

### 检查是否已安装：

1. **打开命令行**
   - Windows：按 `Win + R` → 输入 `cmd` → 回车
   - Mac：打开"终端"（在"应用程序"→"实用工具"里）

2. **输入以下命令并回车**：
   ```bash
   node -v
   ```

3. **判断**：
   - 如果显示版本号（如 `v18.17.0`）→ 已经安装了，跳过这一步
   - 如果显示"命令不存在"或类似的错误 → 需要安装

### 安装 Node.js：

#### Windows 系统：

1. **访问官网**
   - 打开浏览器
   - 访问：https://nodejs.org

2. **下载安装包**
   - 点击 "Download" 按钮（通常是大号的绿色按钮）
   - 会自动下载 `node-vxx.x.x-x64.msi` 或 `.exe` 文件

3. **安装**
   - 双击下载的文件
   - 点击"Next"（下一步）→ 继续点击"Next"
   - 一直点击"Next"，最后点击"Install"
   - 等待安装完成
   - 点击"Finish"

4. **验证安装**
   - 关闭命令行，重新打开
   - 输入 `node -v`，应该显示版本号

#### Mac 系统：

1. **使用 Homebrew 安装（推荐）**
   - 打开终端
   - 复制以下命令并粘贴（按 `Command + V`）：
     ```bash
     /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
     ```
   - 按回车，等待安装完成（需要输入密码）

   - 安装 Node.js：
     ```bash
     brew install node
     ```

2. **或者直接下载安装包**
   - 访问：https://nodejs.org
   - 下载 macOS 安装包
   - 双击安装

## 步骤 4：打开项目文件夹（1分钟）

### Windows 系统：

1. **找到 `client` 文件夹**
   - 应该在你解压的位置（比如桌面）

2. **打开命令行并进入文件夹**
   - **方法 A**：在文件夹空白处，按住 `Shift` 键 → 右键 → 选择"在此处打开命令窗口"或"在此处打开 PowerShell 窗口"
   - **方法 B**：
     - 打开命令行（`Win + R` → `cmd`）
     - 输入以下命令（把路径改成你的实际路径）：
       ```bash
       cd C:\Users\你的用户名\Desktop\client
       ```
     - 按回车

### Mac 系统：

1. **找到 `client` 文件夹**

2. **打开终端并进入文件夹**
   - 打开终端
   - 输入 `cd `（注意有空格）
   - 把 `client` 文件夹拖到终端窗口
   - 按回车

## 步骤 5：安装项目依赖（3-5分钟）

### 在命令行中输入以下命令：

```bash
npm install
```

### 等待过程：
- 会显示很多安装信息
- 可能需要 3-5 分钟
- 等到出现提示符（如 `C:\...` 或 `$`）表示完成

### 如果出错：
- 尝试重新运行 `npm install`
- 或者使用 `npm install --force`

---

## 步骤 6：安装 EAS CLI（1-2分钟）

### 在命令行中输入以下命令：

```bash
npm install -g eas-cli
```

- 1-2 分钟完成

## 步骤 8：构建 APK（5-10分钟）

### 在命令行中输入以下命令：

```bash
eas build --platform android --profile preview
```

### 会提示：

1. **确认构建信息**
   - 会显示项目信息、构建配置等
   - 输入 `y` 然后回车（确认）

2. **开始构建**
   - 会显示 "Build started"
   - 构建过程在云端进行（不需要你的电脑运行）
   - 可以关闭命令行，不影响构建

### 构建时间：
- 通常 5-10 分钟
- 复杂项目可能更久

---

## 步骤 9：下载 APK（2分钟）

构建完成后：

### 方法 1：通过浏览器下载

1. **打开浏览器**
   - Chrome、Edge、Firefox 都可以

2. **访问 Expo 官网**
   - 访问：https://expo.dev
   - 点击右上角"Sign In"

3. **登录**
   - 使用你的账号：zxqyhm@163.com
   - 输入密码登录

4. **找到你的项目**
   - 登录后会看到你的项目列表
   - 找到 "lingo" 项目（如果没有，可能是其他名字）

5. **查看构建记录**
   - 点击项目
   - 点击 "Builds" 标签
   - 找到最新的构建记录
   - 状态应该是 "Finished" 或 "Completed"

6. **下载 APK**
   - 点击构建记录
   - 找到 "Download APK" 按钮
   - 点击下载

### 方法 2：通过命令行下载

如果上面的方法不成功：

1. **回到命令行**
   - 打开命令行
   - 进入 `client` 文件夹

2. **查看构建列表**
   ```bash
   eas build:list
   ```

3. **复制构建 ID**
   - 找到最新的构建记录
   - 复制第一列的 ID（如 `12345678-1234-1234-1234-123456789abc`）

4. **下载**
   ```bash
   eas build:download 复制的ID
   ```

## 步骤 10：安装到手机（2分钟）

### 方法 1：通过数据线安装

1. **连接手机和电脑**
   - 用数据线连接

2. **在手机上启用 USB 调试**
   - 打开手机"设置"
   - 进入"关于手机"
   - 连续点击"版本号"7次（启用开发者模式）
   - 返回"设置"，进入"开发者选项"
   - 打开"USB 调试"

3. **安装 APK**
   - 打开命令行
   - 进入 APK 文件所在目录
   - 输入：
     ```bash
     adb install 文件名.apk
     ```
     - 把 `文件名.apk` 替换成实际的文件名
   - 按回车

4. **安装完成**
   - 显示 "Success" 表示成功
   - 手机上会出现 Lingo 应用图标

### 方法 2：通过文件传输

1. **传输文件到手机**
   - 通过 QQ/微信发送 APK 文件到"文件传输助手"
   - 或通过数据线复制到手机

2. **在手机上安装**
   - 找到 APK 文件
   - 点击安装
   - 如果提示"允许安装未知来源应用"，点击"允许"
   - 等待安装完成

3. **打开应用**
   - 点击桌面上的 Lingo 图标
   - 应用启动！

## ❓ 遇到问题？

### 常见错误及解决：

1. **"npm不是内部或外部命令"**
   - 表示 Node.js 没安装成功
   - 重新安装 Node.js

2. **"eas不是内部或外部命令"**
   - 表示 EAS CLI 没安装成功
   - 重新运行 `npm install -g eas-cli`

3. **"登录失败"**
   - 检查邮箱和密码是否正确
   - 访问 https://expo.dev 确认账号状态

4. **"构建失败"**
   - 查看错误信息
   - 重新构建
   - 或者联系我帮忙

5. **"APK 安装失败"**
   - Android 手机需要开启"允许安装未知来源应用"
   - 设置 → 安全 → 允许未知来源
